# Truth Lens 智能体工作流规范 (V3.0)

## 1. 核心工作流 (Core Workflow)

### 阶段 1：搜索意图提取 (Search Intent Extraction)
*   **输入**: 用户选中的原始文本。
*   **输出**: 3-5 个精准搜索关键词。

### 阶段 2：全维度检索 (Full-Spectrum Retrieval)
*   **输出**: 包含网页正文（Raw Content）的检索结果。

### 阶段 3：信源可信度核查 (Source Credibility Verification) - 【V3.0 新增】
*   **处理**: 
    1. 提取搜索结果的域名或媒体名称。
    2. 对比 `source_credibility.json` 知识库。
    3. **分类**:
        *   **A类 (权威)**: 官方新闻机构（如新华网、人民网、BBC、路透社等）。权重：1.0。
        *   **B类 (可靠)**: 垂直领域专业媒体、知名智库。权重：0.8。
        *   **C类 (中性)**: 一般性新闻站点、大型门户。权重：0.5。
        *   **D类 (低信度)**: 社交媒体、匿名博客、已知传闻高发区。权重：0.2。
*   **目的**: 为后续 AI 判定提供“专家意见”，防止被虚假信源带偏。

### 阶段 4：深度加权比对 (Weighted Evidence Reconciliation)
*   **处理**: LLM 在核查时，会收到每个证据的“信源等级”标注。
*   **逻辑**: 当信源冲突时，AI 必须说明理由并优先采信高等级信源。

## 2. 知识库维护 (Knowledge Base)
*   **文件**: `/lib/data/source_credibility.json`
*   **机制**: 
    *   **静态库**: 预置全球及国内主流权威媒体。
    *   **动态评估**: 若信源不在库中，调用 LLM 进行“即时可信度评估”并临时存入会话上下文。
